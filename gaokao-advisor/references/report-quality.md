# 报告质量与用户体验标准

> **核心立场**：报告读者是真实的高三家长/考生，不是开发者，不是顾问同行。
> 他们打开 PDF 的瞬间，会有一连串疑问、焦虑、信任检验——本文档把这些"用户视角痛点"逐个固化为生成约束。

## 一、用户视角痛点清单（必修）

下表是"打开 PDF 后家长头脑中冒出的真实疑问"，每条对应报告中必须满足的硬约束：

### A. 信任与说服（P0，缺一即不及格）

| 用户疑问 | 必备应答 |
|---------|---------|
| "你怎么知道我孩子合适这所校而不是隔壁那所？" | 主推清单每所校论证段必须含"对比段"：与同档次另 1-2 所校的横向对比（保研率/校区/专业实力/地域差） |
| "为什么是这些主推项而不是更多候选？" | 主推清单后必须有"候选剔除日志"小节：列出宽口候选 30-50 所中被剔除的 5-10 所及理由（数据缺失 / 体检受限 / 单科不达 / 学费超 / 地域被排斥） |
| "今年大小年怎么办？" | 每所主推校必须有"敏感性分析"：给出"若该校今年录取位次上浮 5%/下浮 5% 时，考生位次比值"两个数字，让用户看到决策的稳健性 |
| "冲档落榜进征集，征集还有什么校？" | 第九章征集志愿不能再写模板话，必须用 联网检索 拉本省近 3 年征集志愿院校名单（具体校名 + 缺额专业类型）|

### B. 一目了然（P0，决定家长是否看完）

| 痛点 | 解法 |
|------|------|
| "看完 30 页还是不知道怎么填完整志愿表" | 报告**第一页**新增"志愿表填写顺序卡片"：按本省真实志愿结构完整展开全部填报位；如果山东 96 个、辽宁 112 个等无法单页承载，允许多页，但第一页必须给出冲稳保分区和前 10 个关键位。 |
| "表格列太多在 A4 上挤成糊" | 主推清单从 13 列拆分为**主表 7 列**（序号/院校/专业组/校区/录取数据/录取门槛/数据可信度）+ **附表**（学费/选科/转专业/保研率/分流规则）分两栏放下 |
| "全文字+表格太干" | 必须在第二章"个人画像"和第四章"冲稳保概览"之间插 **1-2 张可视化图**：(1) 考生位次在全省的位置分布图；(2) 主推项的 3 年录取位次趋势线 |
| "保研率/学费/转专业都写'详见招生章程'" | **删除"详见招生章程"的偷懒写法**——除非走完三大入口都拉不到，否则必须给具体数字 + 原文链接 |
| "怎么翻了好几页都是同一句话" | **禁止用重复段落凑页数**：同一长段正文不得机械重复超过 3 次；页数不足时必须补充新的学校论证、风险拆解、数据来源或操作清单。 |

### C. 个性化深度（P1，决定家长觉得"懂我"）

| 痛点 | 解法 |
|------|------|
| "孩子内向却推销售导向专业" | 信息采集阶段必须问"性格倾向"（内向/外向/独立思考型/团队协作型），论证段须呼应 |
| "孩子理想是清华为什么不在冲档" | 信息采集阶段必须问"梦想校（无论分数是否够）"，并在报告"个人画像"段诚实评估梦想校的可达性 + 替代路径 |
| "学费上限有了但没说读研后总成本" | 学费列必须显示"本科 4 年合计 + 是否需读研" 两个数字 |
| "女生推去东北读 4 年" | 信息采集时确认"对气候/性别比例/距家乡距离"的偏好，在论证段呼应 |

### D. 操作可执行性（P1）

| 痛点 | 解法 |
|------|------|
| "不知道怎么报名" | 第十章"重要提醒"必须含"本省志愿系统操作流程"：考生省份的志愿填报系统入口 URL + 时间窗口 + 关键步骤 |
| "征集志愿来不及看 30 页" | 报告末尾新增**"应急一页纸"**：包含 (1) 征集志愿入口 URL + 时间窗口 (2) 兜底校 3 所 + 联系电话 (3) "若 X 月 Y 日前未收到录取通知，立刻做：xxx" |
| "孩子有 3 个梦想方向" | 第七章专业建议须含"剧本 A/B/C"：例如"若坚持医学"/"若坚持北京"/"若坚持低学费"——三个方向的志愿表组合 |

### E. 数据透明度（P1）

| 痛点 | 解法 |
|------|------|
| "你说北邮 25 年 671 分，但朋友 670 也录上了" | 录取数据列必须区分"**院校投档线**（专业组最低）"vs"**专业最低录取分**"两个概念，在表头脚注解释 |
| "5 月生成的报告 6 月出分时怎么用" | 第十章必须含"动态更新机制"小节：(1) 出分后用最新位次重新跑一次 (2) 投档后哪些数据需要替换 (3) 模型可被再次调用更新 |

### F. 报告格式与渲染（P0）

| 问题 | 修复 |
|------|------|
| PDF 没目录页码、没页眉页脚 | 模板 YAML 加 `pagestyle: headings`、`numbersections: true`、`toc: true` |
| 重点信息无视觉层次 | 用 Markdown `**加粗**` + 大号引用块 `> [警示]` 区分；不依赖颜色（保黑白印刷友好）|
| 表格在 A4 溢出 | 主推清单分 7 列 + 附表两栏；用 `pandoc --columns=120` 控制列宽 |
| 字体单一阅读累 | 章节标题用 `Songti SC Bold`，正文用 `Songti SC`，等宽数据用 `Menlo` —— PDF 字体差异化 |

### G. 交付物完整度（P1）

最终交付必须包含：

1. `<姓名>_志愿建议报告_2026.pdf`（主报告）
2. `<姓名>_志愿表填写顺序.pdf`（A4 可打印，大字号；按本省真实填报位完整展开，必要时多页）
3. `<姓名>_志愿建议报告.md`（Markdown 源，家长可改）
4. `<姓名>_应急一页纸.pdf`（征集志愿应急卡片，单页）

**高质量交付不能只给一份 30 页 PDF**——家长在不同场景需要不同长度的物料。

## 二、报告硬指标（生成完成后必查）

报告生成完成时，模型必须自检下列硬指标，任一不达即重写：

| 指标 | 目标 | 自检方法 |
|------|------|---------|
| "建议家长核实"出现次数 | ≤ 3 处（且每处必有具体 URL）| `grep -c "建议.*核实"` |
| "详见招生章程"出现次数 | 0（除非穷尽 4 入口失败，且必须脚注列已尝试路径）| `grep -c "详见招生章程"` |
| 主推校的 N-1 年录取数据完整率 | 100% | 检查每行第 5 列 |
| 主推校都有"对比段" | 100% | 每所校论证段含"相比 XX 校..."字样 |
| 敏感性分析覆盖率 | 100% | 每所主推校论证段含"上浮/下浮"两个假设 |
| 候选剔除日志条目数 | ≥ 5 | 5.5 节（候选剔除日志） |
| 征集志愿章节有具体校名 | 是 | 不能只写"待考试院公布" |
| emoji 出现次数 | 0 | Unicode 码点扫描 |
| 数据源链接数 | ≥ 12 条 | 第十一章"数据来源引用清单" |
| 重复长段落 | 0 处严重重复 | 运行 `scripts/validate_outputs.py`，重复正文超过 3 次必须重写 |

## 三、高质量报告的本质：把"信息"变"决策"

普通报告：列出 30 个校的数据。
高质量报告：让用户在 5 分钟内看懂完整志愿表的冲稳保分区和前几个关键位，并能继续按卡片填满本省全部志愿位。

高质量 PDF 的"读者动线"必须是：

```
封面（30 秒）→ 我是谁，为什么要看这报告
  ↓
志愿表填写顺序卡片（1-3 分钟）→ 先看冲稳保分区、前 10 个关键位，再按页填满全部志愿位
  ↓
冲稳保概览图（1 分钟）→ 让我看明白主推项的分布
  ↓
逐校论证段（10-15 分钟）→ 我可以质疑每一所的合理性
  ↓
退档自查表（2 分钟）→ 排雷
  ↓
应急一页纸（1 分钟）→ 万一出意外怎么办
```

**总耗时控制在 20 分钟内**。超出 20 分钟意味着报告失败——家长会放弃，回去用不可靠的聚合工具。

## 四、文风与气质

高质量报告不是把字写漂亮，是说话像"可信赖的学长或顾问"——

| 不要写 | 改为 |
|-------|------|
| "建议考生综合考虑..." | "我们更推荐 A 校而非 B 校，因为..." |
| "可能存在一定风险" | "去年这所校位次上浮 5%，今年大概率延续——你这分数有 60% 命中率" |
| "请家长务必..." | "这件事 6 月 15 日前必做..." |
| "本报告仅供参考" | （免责声明保留，但正文绝不用"仅供参考"打折表达力） |

模型生成报告时，每段写完都问自己一句：**"如果这是写给我亲外甥的，我会这么含糊吗？"**

## 五、视觉化要求（必出）

PDF 必须含 2-3 张图：

1. **位次分布图**：横轴一分一段表的位次区间，竖轴人数；用红色垂直线标"考生位次"；用绿色阴影标"主推校录取位次区间"
2. **3 年录取位次趋势线图**：8 所主推校 × 3 年的折线图；用红色虚线标"考生位次"作为基线
3. **冲稳保气泡图**（可选）：横轴位次，纵轴学费，气泡大小=保研率，颜色按冲/稳/保分

工具：用 Python matplotlib 生成 PNG 后嵌入 Markdown：

```python
import matplotlib.pyplot as plt
import matplotlib
matplotlib.rcParams['font.sans-serif'] = ['Heiti TC', 'Songti SC']
matplotlib.rcParams['axes.unicode_minus'] = False
```

注意：matplotlib 中文字体必须显式声明，否则 PDF 中显示方块。

## 六、面向高质量交付的 3 条铁律

1. **第一页必须有"答案"**：志愿表填写顺序卡片必须在第 1 页或第 2 页，绝不让用户翻 5 页才看到结论
2. **每个数字必须可被质疑**：报告里的每一个数字都要带脚注链接，能让用户点开后秒级验证
3. **每个建议必须可被执行**：不写"建议关注"，写"6 月 25 日 14:00 登录 X 网址做 Y 事"
